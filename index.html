Guía de Git y GitHub

Aprende a controlar versiones de tu código y colaborar con otros desarrolladores de forma eficiente.

¿Qué es Git?

Creado por Linus Torvalds en 2005 (creador de Linux). Git es un sistema de control de versiones que le permite a los desarrolladores trabajar de forma distribuida y registrar el progreso de un proyecto mientras este evoluciona.

Comandos Básicos de Git

Estos son los comandos fundamentales que usarás en tu día a día trabajando con Git:

# Inicializar un repositorio

$ git init

# Agregar archivos al staging

$ git add .

# Crear un commit

$ git commit -m "primer commit"

# Ver el estado del repositorio

$ git status

# Ver historial de commits

$ git log --oneline

¿Qué es GitHub?

GitHub es un servidor de alojamiento en línea para proyectos basados en Git. Permite el desarrollo de manera remota y proporciona seguridad de la nube creando un entorno centralizado donde reside la "copia principal" del proyecto.

Flujo de Trabajo con GitHub

El flujo típico consiste en crear ramas para nuevas funcionalidades, hacer commits, subir los cambios a GitHub y crear un Pull Request para revisión:

# Conectar con repositorio remoto

$ git remote add origin https://github.com/usuario/repo.git

# Subir cambios

$ git push -u origin main

# Crear una rama

$ git checkout -b nueva-feature

# Fusionar rama

$ git merge nueva-feature

Tips y Buenas Prácticas

  • Haz commits pequeños y frecuentes con mensajes descriptivos.
  • Usa ramas para cada feature o bugfix — nunca trabajes directamente en main.
  • Revisa el diff antes de hacer commit con git diff --staged.
  • Agrega un archivo .gitignore para excluir archivos innecesarios del repositorio.
